# Code Review с CodeRabbit: Роли и интеграция

## Роли
- **CodeRabbit** — архитектурный ревьюер высокого уровня. Проверяет соответствие правилам R1–R9, логику безопасности, тестовое покрытие, ссылается на `KNOWN_ISSUES.md`.
- **Pre‑commit хуки** — синтаксический и форматирующий уровень. Выполняют `ruff`, `detect-secrets`, `architecture_linter.py`, `bandit`.

## Что проверяет CodeRabbit
- Наследование от `BaseAgent` (R1)
- Наличие `@require_ephemeris` (R2)
- Отсутствие прямых `requests` в агентах (R3, KI-001)
- Декораторы аутентификации в веб‑маршрутах (R4)
- Регистрацию в `AGENT_AGENTS` (R5)
- Отсутствие `print` на верхнем уровне (R6)
- Безопасность SQL (R7)
- Жёстко заданные секреты (R8, KI-012)
- Экспорт `run_<agent>` (R9)
- Наличие шести обязательных тестов для новых агентов
- Ссылки на известные проблемы (KI-001, KI-007, KI-011, KI-012)

## Чего CodeRabbit НЕ делает (это делают pre‑commit)
- Форматирование кода (Ruff)
- AST‑валидацию
- Базовую проверку секретов (detect-secrets)
- Статический анализ импортов (architecture_linter.py)

## Блокировка PR
- **P1 нарушения** (R1, R2, R3, R5, R8, R9, KI-001/007/012) → **request changes**, слияние заблокировано.
- **S1–S3 мягкие правила** (docstring, print, etc.) → только предупреждения.

## Обновление правил
Правила пересматриваются **ежеквартально** (синхронизировано с ADR из `ARCHITECTURE.md`).

## Как добавить новый агент и пройти ревью
1. Скопируй `agents/_impl/_template_agent.py`.
2. Заполни TODO, добавь регистрацию в `gitagent_registry.py`.
3. Создай `tests/test_<agent>.py` с шестью тестами.
4. Запуши PR — CodeRabbit проверит автоматически.
